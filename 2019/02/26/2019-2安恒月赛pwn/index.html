题目

功能 add delete print

add 时 会申请两个chunk 第一个chunk存放一个print_moon_content指针及第二个chunk的指针，大小为8，用户申请大小不限
delete 时仅仅free，存在uaf

分析

漏洞很明显，存放指针的chunk free后指针不清除，既可以用来泄露堆地址，还可以利用其来被free的unsorted bin泄露libc基址

而且程序提供了system(“cat /home/pwn/flag”)的函数，覆盖即可
